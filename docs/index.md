## 为什么要学 Claude Code 源码

Claude Code 是 Anthropic 官方的 AI 编程 Agent CLI 工具——它不只是"一个聊天机器人"，而是一个**完整的 AI Agent 运行时**：拥有工具系统、权限系统、记忆系统、多 Agent 协调、流式执行引擎、上下文压缩等生产级子系统。

读懂它的源码，你将获得：

| 维度 | 价值 |
|------|------|
| **求职竞争力** | AI Agent 是 2025-2026 最热赛道，能深度拆解一个生产级 Agent 的候选人极度稀缺 |
| **AI 工程能力** | 理解 Prompt Cache、流式工具执行、上下文压缩等 LLM 工程核心难题的工业级解法 |
| **Agent 开发** | 掌握 Async Generator 驱动循环、工具注册工厂、权限决策链、多 Agent 编排等 Agent 架构范式 |
| **AI 编程架构** | 学习如何在 CLI 场景下用 React/Ink 构建 TUI、如何管理有状态的长会话交互 |

## 学习路线图

本站遵循 **"认识它 → 看它跑 → 懂核心能力 → 懂安全机制 → 懂高级特性 → 逐文件吃透 → 提炼模式 → 实战改造"** 的学习路径：

```
第一阶段：建立全局认知 (Ch1-Ch3)
├── Ch1  前置认知 — Claude Code 是什么、AI Agent 本质、技术栈
├── Ch2  架构总览 — 12 个子系统的高层概览
└── Ch3  全链路走读 — 一次用户输入的完整执行流程源码逐行走读

第二阶段：理解核心能力 (Ch4-Ch7)
├── Ch4  API 通信层 — 流式请求、Prompt Cache、Thinking 配置
├── Ch5  System Prompt — Section 机制、优先级覆盖、缓存策略
├── Ch6  记忆系统 — CLAUDE.md + 记忆目录双层架构
└── Ch7  工具系统 — 54+ 工具注册、执行管道、并发调度

第三阶段：理解安全机制 (Ch8-Ch9)
├── Ch8  权限模式 — 7 种模式、YOLO 分类器、bypass-immune
└── Ch9  沙盒安全 — bwrap/sandbox-exec、文件系统隔离、逃逸防护

第四阶段：理解高级特性 (Ch10-Ch13)
├── Ch10 多 Agent — 子 Agent / 协调器 / 群组三种编排
├── Ch11 压缩管理 — 5 级压缩级联、Token 预算
├── Ch12 Hook/插件/技能 — 拦截机制、功能包、用户命令
└── Ch13 斜杠命令 — 命令注册与分发

第五阶段：逐文件覆盖 (Ch14)
└── Ch14 逐目录源码解析 — src/ 下 38 个子目录、20+ 顶级文件

第六阶段：提炼升华 (Ch15-Ch17)
├── Ch15 设计模式 — 8 大架构模式的抽象提炼
├── Ch16 FAQ — 源码级答疑
└── Ch17 二次开发 — 自定义工具/权限/记忆/Hook/技能
```

::: warning 关于源码版本
本站解析的源码来自 [dgai5016/Claude-Code](https://github.com/dgai5016/Claude-Code) 仓库，版本号为 `999.0.0-restored`。该仓库描述为 "Restored Claude Code source tree reconstructed from source maps"，即**从 source map 反编译恢复的工作区**，并非 Anthropic 官方原始源码。

这意味着：
- 部分代码可能不完整，存在缺失的导入或未解析的模块
- 变量名可能不完全准确（反编译过程中可能有偏差）
- 行号可能有偏移
- 部分功能可能受 feature flag 门控而在反编译代码中不可见

尽管如此，该仓库仍然是我们理解 Claude Code 架构和核心逻辑的**最佳可用源码参考**。本站所有解析严格基于该仓库真实代码，不做虚构。
:::
